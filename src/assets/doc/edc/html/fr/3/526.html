Where do I start?

edc is designed for writing pertinent information to be read by users in context.

The starting point is understanding which UI bricks need information.

Information requirements in context

You need to understand what real users need as they use your product and answer that need with minimalism and a progressive reading experience.

User requirements vary throughout their learning curve and in various situations in the product.

One basic guideline can be: do not write for beginner users. Help beginners through onboarding or training.

Use links to provide related types of information. Avoid saturating the user's cognitive space.

Mapping basic needs

To do this, you need some basic product experience and user awareness.

Usually, product experience comes from dialog with a product owner and developers. Map out the first UI bricks and start there so that you get a good feel for how to write for contextualized delivery.

Iterate and fine tune

Publish the first version to your product and get a feel for how users will perceive the contents. Use an agile, iterative approach to writing and tune it as you go.
